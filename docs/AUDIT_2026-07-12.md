# Foncu technical and product audit — 12 July 2026

## Executive finding

The deployed React shell was reachable, but the core fund-list request failed with `TypeError: fetch failed`. This disabled the main search and cascaded into comparison and export failures. The dominant risk was operational coupling: interactive fund-history requests could call TEFAS directly, while the scheduled refresh endpoint itself crashed on a missing constant and relied on a non-standard cron header.

## What was working

- React/Vite application shell, light/dark theme, charts, screeners, portfolio forms, exports, overlap and event page components.
- Express route allowlist and path-traversal checks.
- Supabase service credentials kept in server-only variables; browser code used the public key.
- Initial RLS policies for user portfolios and public market data.
- CSP, HSTS, frame, referrer and permissions headers in Vercel configuration.
- Useful calculation tests for analytics, portfolio valuation, holdings periods, overlap and sync helpers.

## What was broken or high risk

- Production fund discovery could not load; Benchmark and Export exposed the same data-path failure.
- Frontend lockfile did not match its package manifest, so `npm ci` failed.
- Backend tests had 12 failures: sync paths crashed because `HOLDINGS_UNSUPPORTED_REASON` was referenced but not imported.
- Cron tests and implementation disagreed about query-string secrets and the custom `x-vercel-cron` header.
- The documented root schema omitted tables required by current APIs, making fresh setup incomplete.
- `fund-history` could perform many live TEFAS requests during a user interaction, creating timeout and upstream rate-limit exposure.
- Supabase configuration threw during frontend module import even though the product claimed accounts were optional.
- Nine equal-weight tabs, state-only URLs, browser alerts, conflicting design tokens, and a single mobile breakpoint made the interface difficult to navigate and maintain.
- Initial dependency scans found 14 backend vulnerabilities and 9 frontend vulnerabilities, including critical advisories in export tooling.
- The production Vercel account is not authenticated in this local workspace, so environment variables, logs, cron history and a preview deployment still require account access.

## Implemented remediation

- Database-first fund history with explicit TEFAS provenance, refresh timestamp and stale status; interactive requests no longer scrape TEFAS.
- Legacy `funds` fallback during profile-schema drift, safe retriable errors, `/api/health`, request-size limits and burst throttling.
- Bearer-only cron authentication compatible with Vercel, fixed sync crash, and sanitized public errors.
- Canonical migration covering legacy tables, normalized portfolio holdings, ownership RLS, explicit grants, indexes and legacy selection migration.
- Email magic-link and Google-first authentication with GitHub secondary; anonymous analysis remains usable.
- Route-backed navigation, lazy-loaded feature areas, editorial finance theme, WCAG-oriented focus/targets/reduced motion, mobile bottom navigation and overflow containment.
- Portfolio cost basis, valuation, gain/loss, effective exposure, relative risk contribution, concentration observations, methodology and investment-advice disclaimer.
- Removed unpatched SheetJS, upgraded jsPDF/Vite, restored clean installs, added quality CI and expanded Dependabot coverage.
